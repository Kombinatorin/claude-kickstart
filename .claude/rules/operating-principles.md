# Operating Principles

## Purpose
This project uses a modular Claude Code operating model.

## Principles
- Keep the system modular, not monolithic.
- Use short, focused files with one clear purpose each.
- Prefer references over duplicated knowledge.
- Prefer explicit structure over improvisation.
- Use existing templates before inventing new formats.
- Separate planning, implementation, and validation work.
- Log meaningful decisions.
- Archive instead of deleting when historical value exists.

## Execution Standard
- Think before changing.
- Use the right specialist agent for the job.
- Prefer stable workflows over clever one-off hacks.
- Improve the system based on repeated friction, not theoretical completeness.

## Security
- Never place secrets in markdown files.
- Treat tool access as a controlled capability, not a default assumption.
