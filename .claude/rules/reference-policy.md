# Reference Policy

## Priority Order
1. Official Anthropic / Claude documentation
2. Official Claude blog posts
3. Curated secondary setup guides
4. Local project references

## Working Rule
When Claude Code behavior, MCP behavior, settings, subagents, or instruction hierarchy may have changed:
- check official documentation first
- use secondary guides only as support
- avoid acting on uncertain or stale assumptions

## Documentation Practice
- Keep implementation details in `references/`
- Keep control files short
- Link outward instead of repeating long guidance everywhere

## If Uncertainty Exists
- flag the uncertainty
- check the primary source
- then proceed
