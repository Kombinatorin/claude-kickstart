# Beispiele

Anschauungsmaterial — keine echten Daten.

## `plan.html`

Der ursprüngliche Implementierungsplan, mit dem dieses Repo gebaut wurde — als lesbares HTML-Dokument. Wenn dich interessiert, **warum** etwas so strukturiert ist wie es ist, öffne diese Datei im Browser.

```bash
open references/examples/plan.html
```

## `filled-context/`

Eine **fiktive Persona** namens **Sam** (Indie-Developer) zeigt, wie ein vollständig befülltes `context/`-Verzeichnis aussieht. Nutze es als Inspiration, um deine eigenen `context/*.md`-Dateien zu schreiben — oder lass dich von `/onboard` durchfragen.

**Wichtig:** Sam ist ausgedacht. Alle Namen, Tools, Zahlen sind Beispiele.

## Vergleich mit deinem Setup

Wenn du dein Onboarding abgeschlossen hast, mach kurz folgenden Vergleich:

1. Öffne `context/me.md` parallel zu `references/examples/filled-context/me.md`
2. Frage dich: Ist mein Inhalt mindestens so konkret wie Sams?
3. Wenn nein — `/grill-me me-md-vertiefen` starten oder direkt nachschärfen.

Konkrete Beispiele führen zu besseren Claude-Antworten als wolkige Selbstbeschreibungen.
